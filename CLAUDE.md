# 레이싱 트랜스폰더 시스템 설계

## 1. 시스템 개요

차량이 오버헤드 와이어를 통과하는 시점을 비접촉으로 감지하고, 차량별 고유 ID를 식별하여 랩타임 및 구간 가속 기록을 측정하는 시스템.


### 1.1 운용 모드

| 모드 | 구성 | 용도 |
|------|------|------|
| 랩타임 | 베이스 1개 | 피니시라인 통과 기록, 랩타임 측정 |
| 가속 측정 | 베이스 2개 + 메인 컨트롤러 | 두 지점 간 통과 시간 측정 (예: 0-100m 가속) |


### 1.2 구성 요소

```
  [베이스]──[와이어 왕복]──────────────────── ← 갠트리 상부
     │      ↕ 20~50cm
     │    ┌─[트랜스폰더]─┐                   ← 롤후프 꼭대기
     │    │              │
     │    │    차  량     │
     │    └──────────────┘
  ═══════════════════════════════════════════ ← 노면
```

| 구성 요소 | 설명 |
|-----------|------|
| 트랜스폰더 | 차량 롤후프 상단에 고무줄로 고정. 에폭시 포팅 방수. CR2032 구동. |
| 와이어 | 갠트리 양쪽 폴에 걸친 왕복 전선. 측정 지점에 설치. |
| 베이스 | 갠트리 폴에 설치. 와이어 드라이버 + RF 수신/송신 + 픽업 코일을 nRF52840 + TCXO로 통합. |
| 메인 컨트롤러 | 가속 측정 시 사용. 동기 비콘 브로드캐스트 + 결과 수집. nRF52840 + USB → PC. |
| 타이밍 PC | 기록 처리 소프트웨어. 랩타임 모드에서는 베이스에 USB 직결, 가속 모드에서는 메인 컨트롤러에 USB 직결. |


### 1.3 설계 원칙

- 트랜스폰더는 완전 범용: 채널/이벤트 구분 없음, 어떤 이벤트든 그대로 참가
- 이벤트 분리는 베이스 + 타이밍 소프트웨어가 담당
- 도로 시공 없음. 설치/철수 10분.
- 코인셀(CR2032) 하나로 100시간 이상 동작
- 베이스 간 유선 연결 없음. 무선으로 독립 운용.


## 2. 동작 원리


### 2.1 통과 감지 흐름

```
1. 베이스가 와이어에 125kHz AC를 상시 공급

2. 차량이 와이어 밑을 통과
   → 트랜스폰더의 LC 코일에 125kHz 유기
   → LPCOMP 임계 초과 → MCU 웨이크업
   → nRF52840 RF TX (Enhanced ShockBurst, -20dBm)
   → 패킷: [Vehicle ID][Seq][Reserved]
   → ~80μs 간격으로 연속 전송 (~40회 in 3.5ms)
   → 와이어 이탈 → 신호 소실 → MCU 슬립 복귀

3. 베이스가 RF 패킷 수신
   → TCXO 기반 타이머로 μs 정밀도 로컬 타임스탬프 기록
```


### 2.2 랩타임 모드 (베이스 1개)

```
[베이스] ── USB ──→ [타이밍 PC]

1랩째 통과: t1 기록
2랩째 통과: t2 기록
랩타임 = t2 - t1 (동일 타이머, 동기화 불필요)
```


### 2.3 가속 측정 모드 (베이스 2개 + 메인 컨트롤러)

```
[베이스 A] ─ ─ ─ 50~200m ─ ─ ─ [베이스 B]
  와이어 A (시작)                  와이어 B (도착)
     │                               │
     └──(+8dBm RF)──→ [메인] ←──(+8dBm RF)──┘
                        │
                       USB
                        ↓
                   [타이밍 PC]
```

**동기화 프로토콜:**

```
[측정 전 — 1회 동기]
  메인 컨트롤러: sync 패킷 브로드캐스트 (+8dBm)
  → 베이스 A 수신: 로컬 타이머 값 t_sync_A 기록
  → 베이스 B 수신: 로컬 타이머 값 t_sync_B 기록
  → 각자 오프셋 저장

[측정 중]
  차량이 와이어 A 통과
  → 베이스 A: 로컬 타임스탬프 t1 기록
  → RF로 메인에 전송: [Vehicle ID][Base ID][t1][Seq]
     (드랍 시 재전송 — 타임스탬프가 패킷 안에 있으므로 영향 없음)

  차량이 와이어 B 통과
  → 베이스 B: 로컬 타임스탬프 t2 기록
  → RF로 메인에 전송: [Vehicle ID][Base ID][t2][Seq]

[메인 컨트롤러]
  가속 시간 = (t2 - t_sync_B) - (t1 - t_sync_A)
```

**핵심: 타임스탬프는 감지 시점에 베이스 로컬 타이머로 즉시 기록.** 릴레이 패킷이 드랍되든 재전송되든 지연되든 정밀도에 영향 없음.


### 2.4 가속 측정 정밀도

| 오차 요인 | 크기 | 비고 |
|-----------|------|------|
| Sync 패킷 수신 jitter | ±2μs | 두 베이스 간 |
| TCXO 드리프트 (±0.5ppm) | ±0.5μs/초 | 동기 후 프리런 |
| 트랜스폰더 TX jitter | ±5μs | 상쇄됨 (동일 트랜스폰더) |

```
가속 구간 30초 기준:
  sync jitter: ±2μs
  드리프트: ±15μs (0.5μs × 30초)
  합계: ±17μs (28m/s에서 ±0.5mm)
```

±0.1ms (100μs) 정밀도까지 충분히 보장. TCXO 온도 특성에 따라 극한 환경에서 다소 커질 수 있으나, 실용적 가속 측정에 영향 없는 수준.


### 2.5 가속 모드 라디오 스케줄링

베이스는 트랜스폰더 수신(RX)과 메인으로 릴레이(TX)를 시분할로 처리.

```
[평소]        RX 모드 (트랜스폰더 패킷 대기)
[트랜스폰더 감지] → 타임스탬프 기록
[검출 윈도우 종료] → TX 모드 전환 (+8dBm)
                  → 메인에 릴레이 패킷 전송
                  → ACK 수신 또는 재전송
[전송 완료]     → RX 모드 복귀
```

검출 윈도우(~3.5ms)가 끝난 뒤에 릴레이하므로 트랜스폰더 수신을 놓칠 위험 없음. 차 한 대가 동시에 두 와이어를 지날 수 없으므로 타이밍 충돌도 없음.

동기 sync 패킷은 측정 시작 전에 1회만 수신하면 되므로, 측정 중 라디오 스케줄에 영향 없음.


### 2.6 이벤트 분리 (다중 이벤트 동시 운영, 최소 30m 간격)

반경 100~200m 내에서 여러 이벤트가 동시에 운영될 수 있음. 트랜스폰더는 이벤트를 구분하지 않으므로, 베이스 측에서 3단계로 분리.

**1단계 — 낮은 TX 파워 (물리적 거리 제한):**

트랜스폰더 TX 파워를 -20dBm으로 설정. 유효 도달 거리 ~10~20m. 30m 이상 떨어진 베이스는 수신 자체가 어려움.

**2단계 — 픽업 코일 게이트 (물리적 통과 확인):**

베이스에 부착된 픽업 코일이 바로 옆 와이어의 125kHz 신호를 모니터링. 트랜스폰더의 LC 코일이 와이어 근처를 지나면 에너지를 흡수하여 픽업 코일 신호 진폭이 감소. 이 감소가 감지된 시점 전후 ±20ms 윈도우 안에서만 RF 패킷을 유효 처리.

```
픽업 코일 신호 진폭
 ▲
 │────────╲    ╱────────
 │         ╲  ╱
 │          ╲╱  ← 에너지 흡수 딥
 └──────────────────────► 시간
          ↑
     RF 윈도우 열림
```

**3단계 — 소프트웨어 필터링:**

타이밍 소프트웨어에서 이벤트별 등록 차량 목록으로 최종 필터링.

```
운영 흐름:
  대회 전: 타이밍 SW에서 "차량 03 → 이벤트 A" 등록
  이벤트 이동: SW에서 "차량 03 → 이벤트 B"로 변경
  트랜스폰더: 아무것도 안 함
```

**폴백 우선순위:** 2단계(픽업 코일 딥 감지)가 불안정할 경우, 1단계 + 3단계 조합만으로도 분리 가능.


## 3. 트랜스폰더 하드웨어


### 3.1 블록 다이어그램

```
[CR2032] ─→ [nRF52840 모듈]
                  │
                  ├── AIN ←── 엔벨로프 검출 ←── LC 공진 코일
                  │           (다이오드 + RC)     (페라이트 코어
                  │                                + 튜닝 캡)
                  ├── LPCOMP ← 엔벨로프 DC 레벨 감시 (슬립 중 웨이크업)
                  │
                  ├── RADIO → 2.4GHz TX (Enhanced ShockBurst)
                  │
                  ├── GPIO IN ← 홀 센서 (전원 ON/OFF)
                  └── GPIO OUT → LED (상태 표시)
```


### 3.2 주요 부품

| 부품 | 사양 | 역할 |
|------|------|------|
| nRF52840 모듈 | E73-2G4M08S1C 등 | MCU + 2.4GHz Radio |
| 페라이트 코어 코일 | 인덕턴스 TBD (125kHz 튜닝) | 자기장 감지 |
| 튜닝 캡 | C = 1/(4π²f²L) | LC 공진 주파수 맞춤 |
| 검파 다이오드 | BAT54 또는 1N5711 | 엔벨로프 검출 |
| 홀 센서 | DRV5032 (SOT-23) | 외부 자석으로 전원 ON/OFF |
| LED | 0402 또는 0603 | 상태 표시 (포팅 표면으로 투과) |
| CR2032 홀더 | SMD | 전원 |


### 3.3 LC 코일 튜닝

```
공진 주파수 f = 125kHz
코일 인덕턴스 L을 측정 후 C 계산:

  C = 1 / (4π² × f² × L)

예: L = 10mH → C = 162pF
예: L = 4.7mH → C = 345pF
```

Q 팩터 목표: Q ≈ 20~50. 높을수록 감도가 좋지만 대역폭이 좁아지므로 필요 시 병렬 저항으로 Q를 조정.


### 3.4 웨이크업 회로

```
LC 코일 출력
  → 검파 다이오드 (반파 정류)
  → RC 평활 (τ ≈ 100μs)
  → nRF52840 LPCOMP (내장 저전력 비교기)
  → 임계 초과 시 인터럽트 → MCU 웨이크업
```

nRF52840 내장 LPCOMP는 슬립 중에도 ~1μA로 동작. 외부 비교기 불필요.


### 3.5 RF 전송

| 파라미터 | 값 |
|----------|-----|
| 모드 | Enhanced ShockBurst (proprietary, BLE 아님) |
| 클럭 | HFINT (내장 RC, 시작 ~1μs) |
| 데이터레이트 | 2Mbps |
| TX 파워 | -20dBm (프로토타입에서 조정 가능) |
| 채널 | 고정 (예: 채널 80, 2480MHz) |
| 주소 | 5바이트 고정 (예: 0xE7E7E7E7E7) |

HFXO(크리스탈) 대신 HFINT(RC)를 사용하여 웨이크업 지연 ~360μs를 ~1μs로 단축. 주파수 정확도 ±1.7%이지만 proprietary 모드에서 수신부가 허용 가능.


### 3.6 RF 패킷 구조

```
Enhanced ShockBurst 프레임:
┌──────────┬──────────┬─────────────────┬─────┐
│ Preamble │ Address  │    Payload      │ CRC │
│  1B      │  5B      │    3B           │ 2B  │
│ (HW자동) │ (HW자동) │                 │(HW자동)│
└──────────┴──────────┴─────────────────┴─────┘
                       │
                       ├── Vehicle ID (1B): 0x01~0xFF (255대)
                       ├── Sequence  (1B): 0~255 롤링 (유실 감지)
                       └── Reserved  (1B): 향후 확장용
```

Preamble, Address, CRC는 nRF52840 하드웨어가 자동 처리. 펌웨어에서는 3바이트 페이로드만 설정.


### 3.7 동작 시퀀스

```
[슬립] ─── LPCOMP 트리거 ───→ [웨이크업]
  ↑                              │
  │   ┌──────────────────────────┘
  │   │
  │   ├─ HFINT 시작              (~1μs)
  │   ├─ RADIO 설정              (~5μs)
  │   ├─ TX ramp-up              (~40μs)
  │   ├─ 패킷 송출               (~20μs)
  │   ├─ 반복 (LPCOMP 활성 동안)
  │   │    ~80μs 간격 연속 전송
  │   │    검출 영역 ~10cm → ~3.5ms 동안 ~40회
  │   │
  │   └─ LPCOMP 해제 → TX 중단
  │
  └──── [슬립 복귀]
```


### 3.8 전력 추정

| 상태 | 전류 | 비고 |
|------|------|------|
| Deep sleep + LPCOMP | ~3μA | 99.99% 시간 |
| TX burst (-20dBm) | ~5mA | ~3.5ms per crossing |

```
랩타임 60초 기준:
I_avg = 3μA + (5mA × 3.5ms / 60s) = 3μA + 0.29μA ≈ 3.3μA
CR2032 (220mAh) → 220,000μAh / 3.3μA ≈ 66,000시간
```

자연 방전(연 ~1%)이 수명 제한 요인. 100시간 요구사항 대비 수천 배 마진.


### 3.9 외장

```
  ┌─────────────────────────┐
  │  ┌───┐ ┌─────────────┐  │
  │  │LED│ │ nRF52 모듈   │  │
  │  └───┘ └─────────────┘  │
  │  ┌────┐ ┌────────────┐  │  ~40 × 30 × 15mm
  │  │Hall│ │ CR2032      │  │  우레탄 포팅
  │  └────┘ └────────────┘  │
  │  ┌──────────────────┐   │
  │  │ 페라이트 코일      │   │
  │  └──────────────────┘   │
  └─────────────────────────┘
```

- PCB + 부품 + 코인셀을 실리콘 몰드에 배치 후 우레탄 포팅제(예: Polytek Poly 74-20)로 충전 경화
- 물리적 개구부 없음 → IP68 방수
- LED는 포팅 표면 가까이 배치 → 빛 투과
- 홀 센서도 표면 가까이 → 외부 자석 터치로 전원 ON/OFF
- 롤후프 파이프에 고무줄로 고정


## 4. 베이스 하드웨어 (드라이버 + 수신부 통합)

nRF52840 + TCXO 하나가 드라이버, RF RX/TX, 픽업 코일 게이트를 모두 담당.


### 4.1 블록 다이어그램

```
[USB-C PD 충전기]
      │
[USB-C 커넥터 + CH224K] → 12V
      │
      ├──→ [H-bridge] (IR2104+FET×2) → [와이어 왕복] + 튜닝 캡
      │
      └──→ [페라이트 비드] → [3.3V LDO] → [nRF52840] ← [TCXO 16MHz]
                                              │
                              ┌────────────────┼────────────────┐
                              │                │                │
                         TIMER/PWM          RADIO           ADC/LPCOMP
                         125kHz 구형파       ESB RX/TX        픽업 코일
                              ↓                ↓             모니터링
                           GPIO×2        트랜스폰더 수신        ↓
                              ↓           메인 릴레이 TX       AIN
                           H-bridge       sync RX              ↓
                           제어            (+8dBm)         [픽업 코일]
                                              │           (와이어 옆)
                                         USB/UART
                                              ↓
                                       [타이밍 PC]
                                    (랩타임 모드 시 직결)
```

TIMER, RADIO, ADC는 독립 하드웨어 페리페럴이므로 동시 동작에 간섭 없음.


### 4.2 TCXO

| 파라미터 | 값 |
|----------|-----|
| 부품 예시 | Abracon ASTX-H11-16.000MHz |
| 주파수 정확도 | ±0.5ppm |
| 패키지 | 3.2 × 2.5mm SMD |
| 단가 | ~$1~2 |

nRF52840의 HFXO 입력에 연결. 이 클럭으로 타이머를 구동하여 μs 정밀도 타임스탬프 생성.

랩타임 모드에서는 TCXO 정밀도가 필요 없지만 (같은 클럭으로 두 번 찍으니까), 가속 측정 모드에서 두 베이스 간 클럭 드리프트를 최소화하는 데 필수.


### 4.3 와이어

```
  폴 ──────────────────────────── 폴
     ────────────────────────────   ← 왕복 와이어 (수 cm 간격)
     ← ─ ─ ─ 트랙 폭 5~7m ─ ─ →
```

- 갠트리 양쪽 폴 사이에 왕복으로 걸침
- 간격: 수 cm (왕복 와이어 간)
- 재질: 일반 절연 전선 (AWG 18~22)
- 길이: 트랙 폭 × 왕복 = 10~14m
- 배너 걸듯이 설치/철수


### 4.4 전원부

```
[USB-C PD 충전기] → [USB-C 커넥터]
                        │
                    [CH224K]
                    (12V PD 트리거, 저항 1개로 전압 설정)
                        │
                     12V 출력
                        │
          ┌─────────────┴─────────────┐
          │                           │
    [H-bridge]                  [페라이트 비드]
    (IR2104+FET)                      │
          │                     [3.3V LDO]
          ↓                     (AMS1117-3.3)
       와이어                         │
                                 nRF52840
```

- CH224K: ESSOP-10 패키지, 전압 설정 저항 1개로 12V 트리거
- USB PD 12V 최소 1.5A(18W) 지원 → 실소비 5W 미만으로 마진 충분
- 페라이트 비드: H-bridge 스위칭 노이즈가 nRF52840 라디오에 유입되는 것을 차단


### 4.5 드라이버 회로

```
nRF52840 TIMER/PWM (125kHz 구형파)
  → GPIO 2핀
  → H-bridge (IR2104 + N-ch MOSFET ×2)
  → LC 공진 (와이어 인덕턴스 + 튜닝 캡)
  → 와이어에 정현파 전류
```

- 와이어 인덕턴스 측정 후 C 값으로 125kHz 공진 맞춤
- LC 공진이므로 적은 입력 전력으로 강한 전류 생성
- 목표 전류: 0.5~2A (프로토타입에서 조정)


### 4.6 RF 설정

| 파라미터 | 값 (트랜스폰더 수신) | 값 (메인 릴레이) |
|----------|---------------------|-----------------|
| 모드 | ESB RX | ESB TX |
| 채널 | 채널 80 | 채널 40 (분리) |
| 주소 | 0xE7E7E7E7E7 | 0xA1A1A1A1A1 |
| TX 파워 | - | +8dBm |

트랜스폰더 수신 채널과 메인 릴레이 채널을 분리하여 간섭 방지.


### 4.7 릴레이 패킷 구조 (베이스 → 메인)

```
┌──────────┬──────────┬──────────────────────────────┬─────┐
│ Preamble │ Address  │          Payload             │ CRC │
│  1B      │  5B      │          8B                  │ 2B  │
│ (HW자동) │ (HW자동) │                              │(HW자동)│
└──────────┴──────────┴──────────────────────────────┴─────┘
                       │
                       ├── Vehicle ID    (1B)
                       ├── Base ID       (1B): 베이스 식별
                       ├── Timestamp     (4B): TCXO 기반 로컬 타임스탬프 (μs)
                       ├── RSSI          (1B): 트랜스폰더 수신 신호 강도
                       └── Sequence      (1B): 릴레이 패킷 시퀀스
```


### 4.8 픽업 코일 게이트

```
[픽업 코일] (와이어 바로 옆 수 cm에 배치)
  → 엔벨로프 검출 (다이오드 + RC)
  → nRF52840 ADC (연속 샘플링)
  → 소프트웨어: 진폭 딥 감지
  → 딥 전후 ±20ms 윈도우 내 RF만 유효 처리
```


## 5. 메인 컨트롤러 하드웨어 (가속 측정 모드 전용)

랩타임 모드에서는 불필요. 가속 측정 시에만 사용.


### 5.1 블록 다이어그램

```
[nRF52840 모듈]
      │
      ├── RADIO: ESB TX (sync 비콘) / ESB RX (릴레이 수신)
      │          채널 40, 주소 0xA1A1A1A1A1
      │          TX 파워: +8dBm
      │
      └── USB → [타이밍 PC]
```

별도 TCXO 불필요 (자체 타이밍이 아니라 베이스들의 타임스탬프를 수집/계산만 함).


### 5.2 Sync 비콘 패킷

```
┌──────────┬──────────┬───────────┬─────┐
│ Preamble │ Address  │ Payload   │ CRC │
│  1B      │  5B      │ 1B        │ 2B  │
│ (HW자동) │ (HW자동) │           │(HW자동)│
└──────────┴──────────┴───────────┴─────┘
                       │
                       └── Sync ID (1B): 동기화 식별자
```

측정 시작 전 1회 브로드캐스트. 모든 베이스가 이 패킷 수신 시점을 자기 로컬 타이머 기준으로 기록.


### 5.3 프로토타입

nRF52840-DK 1개 + USB → PC. 추가 회로 없음.


## 6. 타이밍 정밀도 분석


### 6.1 트랜스폰더 → 베이스 지연 체인

```
자기장 감지 → LPCOMP 트리거    ~1μs
MCU 웨이크업                   ~5μs
HFINT 시작                     ~1μs
Radio ramp-up                  ~40μs
패킷 전송 (3B @ 2Mbps)         ~20μs
RF 전파 (2~3m)                 ~10ns
베이스 수신 처리                ~10μs
──────────────────────────────────
총 지연: ~77μs
지연 변동 (jitter): ±5μs
```


### 6.2 랩타임 정밀도 (단일 베이스)

```
고정 지연(~77μs) 상쇄 → jitter만 영향
±5μs = 28m/s 기준 ±0.14mm
```


### 6.3 가속 측정 정밀도 (듀얼 베이스)

```
Sync 패킷 수신 jitter: ±2μs
TCXO 드리프트 (30초): ±15μs
합계: ±17μs = 28m/s 기준 ±0.5mm
```

극한 온도 환경에서 TCXO 특성이 다소 변할 수 있으나, ±0.1ms 이내 실용 정밀도 확보.


### 6.4 검출 신뢰도

```
와이어 유효 자기장 범위: ~10~20cm
28m/s 기준:
  10cm → ~3.5ms → ~40회 패킷 전송
  20cm → ~7ms  → ~80회 패킷 전송
40~80회 중 1회만 수신 성공하면 됨
```

## 8. 프로토타입 단계


### Phase 1 — 자기 결합 검증 (최우선, 프로젝트 성패 결정)

- nRF52840-DK 하나로 125kHz PWM → H-bridge → 와이어 구동
- 소형 페라이트 코일을 20~50cm 거리에 배치
- 오실로스코프로 코일 유기전압 측정
- 목표: 비교기 감지 가능한 수준 (수십 mV 이상)
- 실패 시: 와이어 전류 증가 / 코일 크기 증가 / 턴 수 증가로 재시도


### Phase 2 — 트랜스폰더 검증

- nRF52840-DK + 브레드보드에 LC 코일 + 엔벨로프 검출 회로
- LPCOMP 웨이크업 동작 확인
- 웨이크업 → ESB TX 패킷 전송 확인
- 전류 소비 측정 (슬립 + 버스트)
- 코일 파라미터(인덕턴스, Q, 튜닝 캡) 최적화


### Phase 3 — 베이스 통합 (랩타임 모드)

- nRF52840-DK 하나에 드라이버(PWM + H-bridge) + RF 수신(ESB RX) 동시 동작
- 픽업 코일 딥 감지 검증
- 와이어 밑 수동 통과 테스트 (손으로 트랜스폰더 이동)
- 타임스탬프 기록 → PC USB 전송 확인


### Phase 4 — 가속 측정 모드 검증

- nRF52840-DK 3개: 베이스 A, 베이스 B, 메인 컨트롤러
- sync 비콘 브로드캐스트 + 수신 테스트
- 두 베이스 간 타임스탬프 오차 측정 (동일 트랜스폰더를 동시에 두 베이스 앞에서 트리거)
- 릴레이 TX/RX + 드랍 재전송 검증
- TCXO 드리프트 실측 (DK 단계에서는 기판 크리스탈로 테스트, 정밀도 한계 확인)


### Phase 5 — 실차 테스트

- 트랜스폰더를 롤후프에 장착
- 저속 → 고속 주행 테스트
- EMI 환경에서 검출 신뢰도 확인 (전동차 모터/인버터)
- 반복 통과 시 타이밍 일관성 검증
- 가속 측정 모드 실차 검증
- 우천 테스트


### Phase 6 — 커스텀 PCB + 포팅

- Phase 1~5에서 확정된 파라미터로 PCB 설계 (KiCad)
- 트랜스폰더: 소형 PCB + 우레탄 포팅
- 베이스: TCXO 포함, 방수 케이스 내장
- 메인 컨트롤러: USB 동글 형태
- 수량 제작
- 타이밍 소프트웨어 연동
