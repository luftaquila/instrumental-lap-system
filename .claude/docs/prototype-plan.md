# 프로토타입 단계 및 리스크

## Phase 1 — 자기 결합 검증 (최우선, 프로젝트 성패 결정)

- nRF52840-DK 하나로 125kHz PWM → H-bridge → 와이어 구동
- 소형 페라이트 코일을 20~50cm 거리에 배치
- 오실로스코프로 코일 유기전압 측정
- 목표: 비교기 감지 가능한 수준 (수십 mV 이상)
- 실패 시: 와이어 전류 증가 / 코일 크기 증가 / 턴 수 증가로 재시도

## Phase 2 — 트랜스폰더 검증

- nRF52840-DK + 브레드보드에 LC 코일 + 엔벨로프 검출 회로
- LPCOMP 웨이크업 동작 확인
- 웨이크업 → ESB TX 패킷 전송 확인
- 전류 소비 측정 (슬립 + 버스트)
- 코일 파라미터(인덕턴스, Q, 튜닝 캡) 최적화

## Phase 3 — 베이스 통합 (랩타임 모드)

- nRF52840-DK 하나에 드라이버(PWM + H-bridge) + RF 수신(ESB RX) 동시 동작
- 픽업 코일 딥 감지 검증
- 와이어 밑 수동 통과 테스트 (손으로 트랜스폰더 이동)
- 타임스탬프 기록 → PC USB 전송 확인

## Phase 4 — 가속 측정 모드 검증

- nRF52840-DK 3개: 베이스 A, 베이스 B, 메인 컨트롤러
- sync 비콘 브로드캐스트 + 수신 테스트
- 두 베이스 간 타임스탬프 오차 측정 (동일 트랜스폰더를 동시에 두 베이스 앞에서 트리거)
- 릴레이 TX/RX + 드랍 재전송 검증
- TCXO 드리프트 실측 (DK 단계에서는 기판 크리스탈로 테스트, 정밀도 한계 확인)

## Phase 5 — 실차 테스트

- 트랜스폰더를 롤후프에 장착
- 저속 → 고속 주행 테스트
- EMI 환경에서 검출 신뢰도 확인 (전동차 모터/인버터)
- 반복 통과 시 타이밍 일관성 검증
- 가속 측정 모드 실차 검증
- 우천 테스트

## Phase 6 — 커스텀 PCB + 포팅

- Phase 1~5에서 확정된 파라미터로 PCB 설계 (KiCad)
- 트랜스폰더: 소형 PCB + 우레탄 포팅
- 베이스: TCXO 포함, 방수 케이스 내장
- 메인 컨트롤러: USB 동글 형태
- 수량 제작
- 타이밍 소프트웨어 연동

## 리스크 및 폴백

| 리스크 | 영향 | 폴백 |
|--------|------|------|
| Phase 1에서 유기전압 부족 | 프로젝트 근본 위협 | 와이어 전류 증가, 코일 크기/턴 수 증가 |
| EMI로 오트리거 | 오탐 발생 | LPCOMP 임계값 상향, 소프트웨어 디바운스 |
| 픽업 코일 딥 감지 불안정 | 이벤트 분리 약화 | TX 파워 제한(-20dBm) + SW 필터링으로 대체 |
| 우레탄 포팅 후 코일 감도 변화 | 감지 실패 | 포팅 전후 튜닝 값 보정, Q팩터 마진 확보 |
| CR2032 전압 강하 시 동작 불안정 | 수명 단축 | nRF52840은 1.7V까지 동작, 마진 충분 |
| 2.4GHz 혼잡 (Wi-Fi 등) | 패킷 유실 | 3.5ms 윈도우 내 ~40회 전송, 1회만 성공하면 됨 |
| TCXO 온도 차이로 드리프트 증가 | 가속 측정 오차 증가 | sync 주기를 늘리거나, 측정 직전 재동기 |
| 릴레이 패킷 연속 드랍 | 기록 유실 | 베이스가 로컬 저장 후 재전송 (최대 10회) |
| 가속 모드 라디오 시분할 타이밍 | 트랜스폰더 수신 누락 | 검출 윈도우 종료 후에만 릴레이 TX 수행 |
