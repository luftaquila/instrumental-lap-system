# 베이스 하드웨어 (드라이버 + 수신부 통합)

nRF52840 + TCXO 하나가 드라이버, RF RX/TX, 픽업 코일 게이트를 모두 담당.

## 블록 다이어그램

```
[USB-C PD 충전기]
      │
[USB-C 커넥터 + CH224K] → 12V
      │
      ├──→ [H-bridge] (IR2104+FET x 2) → [와이어 왕복] + 튜닝 캡
      │
      └──→ [페라이트 비드] → [3.3V LDO] → [nRF52840] ← [TCXO 16MHz]
                                              │
                              ┌────────────────┼────────────────┐
                              │                │                │
                         TIMER/PWM          RADIO           ADC/LPCOMP
                         125kHz 구형파       ESB RX/TX        픽업 코일
                              ↓                ↓             모니터링
                           GPIO x 2      트랜스폰더 수신        ↓
                              ↓           메인 릴레이 TX       AIN
                           H-bridge       sync RX              ↓
                           제어            (+8dBm)         [픽업 코일]
                                              │           (와이어 옆)
                                         USB/UART
                                              ↓
                                       [타이밍 PC]
                                    (랩타임 모드 시 직결)
```

TIMER, RADIO, ADC는 독립 하드웨어 페리페럴이므로 동시 동작에 간섭 없음.

## TCXO

| 파라미터 | 값 |
|----------|-----|
| 부품 예시 | Abracon ASTX-H11-16.000MHz |
| 주파수 정확도 | ±0.5ppm |
| 패키지 | 3.2 x 2.5mm SMD |
| 단가 | ~$1~2 |

nRF52840의 HFXO 입력에 연결. 이 클럭으로 타이머를 구동하여 us 정밀도 타임스탬프 생성.
랩타임 모드에서는 TCXO 정밀도 불필요 (동일 클럭으로 두 번 찍으니까), 가속 측정 모드에서 두 베이스 간 드리프트 최소화에 필수.

## 와이어

```
  폴 ──────────────────────────── 폴
     ────────────────────────────   ← 왕복 와이어 (수 cm 간격)
     ← ─ ─ ─ 트랙 폭 5~7m ─ ─ →
```

- 갠트리 양쪽 폴 사이에 왕복으로 걸침
- 간격: 수 cm (왕복 와이어 간)
- 재질: 일반 절연 전선 (AWG 18~22)
- 길이: 트랙 폭 x 왕복 = 10~14m

## 전원부

```
[USB-C PD 충전기] → [USB-C 커넥터]
                        │
                    [CH224K] (ESSOP-10, 저항 1개로 12V 트리거)
                        │
                     12V 출력
                        │
          ┌─────────────┴─────────────┐
          │                           │
    [H-bridge]                  [페라이트 비드]
    (IR2104+FET)                      │
          │                     [3.3V LDO] (AMS1117-3.3)
          ↓                           │
       와이어                      nRF52840
```

- USB PD 12V 최소 1.5A(18W) 지원 → 실소비 5W 미만으로 마진 충분
- 페라이트 비드: H-bridge 스위칭 노이즈가 nRF52840 라디오에 유입되는 것을 차단

## 드라이버 회로

```
nRF52840 TIMER/PWM (125kHz 구형파)
  → GPIO 2핀
  → H-bridge (IR2104 + N-ch MOSFET x 2)
  → LC 공진 (와이어 인덕턴스 + 튜닝 캡)
  → 와이어에 정현파 전류
```

- 와이어 인덕턴스 측정 후 C 값으로 125kHz 공진 맞춤
- LC 공진이므로 적은 입력 전력으로 강한 전류 생성
- 목표 전류: 0.5~2A (프로토타입에서 조정)

## 픽업 코일 게이트

```
[픽업 코일] (와이어 바로 옆 수 cm에 배치)
  → 엔벨로프 검출 (다이오드 + RC)
  → nRF52840 ADC (연속 샘플링)
  → 소프트웨어: 진폭 딥 감지
  → 딥 전후 ±20ms 윈도우 내 RF만 유효 처리
```
