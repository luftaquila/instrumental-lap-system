# 통과 감지 흐름 및 동작 모드

## 감지 흐름

```
1. 베이스가 와이어에 125kHz AC를 상시 공급

2. 차량이 와이어 밑을 통과
   → 트랜스폰더의 LC 코일에 125kHz 유기
   → LPCOMP 임계 초과 → MCU 웨이크업
   → nRF52840 RF TX (Enhanced ShockBurst, -20dBm)
   → 패킷: [Vehicle ID][Seq][Reserved]
   → ~80us 간격으로 연속 전송 (~40회 in 3.5ms)
   → 와이어 이탈 → 신호 소실 → MCU 슬립 복귀

3. 베이스가 RF 패킷 수신
   → TCXO 기반 타이머로 us 정밀도 로컬 타임스탬프 기록
```

## 랩타임 모드 (베이스 1개)

```
[베이스] ── USB ──→ [타이밍 PC]

1랩째 통과: t1 기록
2랩째 통과: t2 기록
랩타임 = t2 - t1 (동일 타이머, 동기화 불필요)
```

## 가속 측정 모드 (베이스 2개 + 메인 컨트롤러)

```
[베이스 A] ─ ─ ─ 50~200m ─ ─ ─ [베이스 B]
  와이어 A (시작)                  와이어 B (도착)
     │                               │
     └──(+8dBm RF)──→ [메인] ←──(+8dBm RF)──┘
                        │
                       USB
                        ↓
                   [타이밍 PC]
```

### 동기화 프로토콜

```
[측정 전 — 1회 동기]
  메인 컨트롤러: sync 패킷 브로드캐스트 (+8dBm)
  → 베이스 A 수신: 로컬 타이머 값 t_sync_A 기록
  → 베이스 B 수신: 로컬 타이머 값 t_sync_B 기록
  → 각자 오프셋 저장

[측정 중]
  차량이 와이어 A 통과
  → 베이스 A: 로컬 타임스탬프 t1 기록
  → RF로 메인에 전송: [Vehicle ID][Base ID][t1][Seq]
     (드랍 시 재전송 — 타임스탬프가 패킷 안에 있으므로 영향 없음)

  차량이 와이어 B 통과
  → 베이스 B: 로컬 타임스탬프 t2 기록
  → RF로 메인에 전송: [Vehicle ID][Base ID][t2][Seq]

[메인 컨트롤러]
  가속 시간 = (t2 - t_sync_B) - (t1 - t_sync_A)
```

**핵심: 타임스탬프는 감지 시점에 베이스 로컬 타이머로 즉시 기록.** 릴레이 패킷이 드랍되든 재전송되든 지연되든 정밀도에 영향 없음.

### 가속 측정 정밀도

| 오차 요인 | 크기 | 비고 |
|-----------|------|------|
| Sync 패킷 수신 jitter | ±2us | 두 베이스 간 |
| TCXO 드리프트 (±0.5ppm) | ±0.5us/초 | 동기 후 프리런 |
| 트랜스폰더 TX jitter | ±5us | 상쇄됨 (동일 트랜스폰더) |

가속 구간 30초 기준: sync jitter ±2us + 드리프트 ±15us = 합계 ±17us (28m/s에서 ±0.5mm)

## 가속 모드 라디오 스케줄링

베이스는 트랜스폰더 수신(RX)과 메인으로 릴레이(TX)를 시분할로 처리.

```
[평소]        RX 모드 (트랜스폰더 패킷 대기)
[트랜스폰더 감지] → 타임스탬프 기록
[검출 윈도우 종료] → TX 모드 전환 (+8dBm)
                  → 메인에 릴레이 패킷 전송
                  → ACK 수신 또는 재전송
[전송 완료]     → RX 모드 복귀
```

검출 윈도우(~3.5ms)가 끝난 뒤에 릴레이하므로 트랜스폰더 수신을 놓칠 위험 없음. 차 한 대가 동시에 두 와이어를 지날 수 없으므로 타이밍 충돌도 없음.

## 이벤트 분리 (다중 이벤트 동시 운영, 최소 30m 간격)

트랜스폰더는 이벤트를 구분하지 않으므로 베이스 측에서 3단계로 분리:

**1단계 — 낮은 TX 파워:** 트랜스폰더 -20dBm → 유효 도달 ~10~20m. 30m 이상 베이스는 수신 불가.

**2단계 — 픽업 코일 게이트:** 베이스 픽업 코일이 125kHz 신호 모니터링. 트랜스폰더 통과 시 에너지 흡수 → 진폭 딥. 딥 전후 ±20ms 윈도우 안에서만 RF 유효 처리.

```
픽업 코일 신호 진폭
 ▲
 │────────╲    ╱────────
 │         ╲  ╱
 │          ╲╱  ← 에너지 흡수 딥
 └──────────────────────► 시간
          ↑
     RF 윈도우 열림
```

**3단계 — 소프트웨어 필터링:** 타이밍 SW에서 이벤트별 등록 차량 목록으로 최종 필터링.

**폴백:** 2단계 불안정 시, 1단계 + 3단계 조합으로 분리 가능.
