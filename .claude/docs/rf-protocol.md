# RF 프로토콜 및 패킷 구조

## 채널 배치

| 링크 | 채널 | 주소 | TX 파워 | 용도 |
|------|------|------|---------|------|
| 트랜스폰더 → 베이스 | CH80 (2480MHz) | 0xE7E7E7E7E7 | -20dBm | 통과 감지 |
| 베이스 → 메인 | CH40 | 0xA1A1A1A1A1 | +8dBm | 릴레이/sync |

공통: Enhanced ShockBurst (proprietary, BLE 아님), 2Mbps, CRC 2B

## 트랜스폰더 RF 설정

| 파라미터 | 값 |
|----------|-----|
| 클럭 | HFINT (내장 RC, 시작 ~1us) |
| 데이터레이트 | 2Mbps |
| TX 파워 | -20dBm (프로토타입에서 조정 가능) |

HFXO 대신 HFINT 사용하여 웨이크업 지연 ~360us를 ~1us로 단축. 주파수 정확도 ±1.7%이지만 proprietary 모드에서 수신부가 허용 가능.

## 트랜스폰더 패킷 (3B payload)

```
ESB 프레임:
┌──────────┬──────────┬─────────────────┬─────┐
│ Preamble │ Address  │    Payload      │ CRC │
│  1B      │  5B      │    3B           │ 2B  │
│ (HW자동) │ (HW자동) │                 │(HW자동)│
└──────────┴──────────┴─────────────────┴─────┘
                       │
                       ├── Vehicle ID (1B): 0x01~0xFF (255대)
                       ├── Sequence  (1B): 0~255 롤링 (유실 감지)
                       └── Reserved  (1B): 향후 확장용
```

Preamble, Address, CRC는 nRF52840 하드웨어가 자동 처리. 펌웨어에서는 3B 페이로드만 설정.

## 릴레이 패킷: 베이스 → 메인 (8B payload)

```
┌──────────┬──────────┬──────────────────────────────┬─────┐
│ Preamble │ Address  │          Payload             │ CRC │
│  1B      │  5B      │          8B                  │ 2B  │
└──────────┴──────────┴──────────────────────────────┴─────┘
                       │
                       ├── Vehicle ID    (1B)
                       ├── Base ID       (1B): 베이스 식별
                       ├── Timestamp     (4B): TCXO 기반 로컬 타임스탬프 (us)
                       ├── RSSI          (1B): 트랜스폰더 수신 신호 강도
                       └── Sequence      (1B): 릴레이 패킷 시퀀스
```

## Sync 비콘 패킷: 메인 → 베이스 (1B payload)

```
┌──────────┬──────────┬───────────┬─────┐
│ Preamble │ Address  │ Payload   │ CRC │
│  1B      │  5B      │ 1B        │ 2B  │
└──────────┴──────────┴───────────┴─────┘
                       │
                       └── Sync ID (1B): 동기화 식별자
```

측정 시작 전 1회 브로드캐스트. 모든 베이스가 수신 시점을 로컬 타이머 기준으로 기록.
